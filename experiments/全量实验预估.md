## 1. 方法说明:未校准的(模型,域)组合如何估算

只有 deepseek-v4-flash 和 deepseek-v4-pro 在三个域都有较可靠的10任务校准均值。其余10个Qwen模型
只在airline域跑过1个真实任务(任务2验证),retail/telecom域没有直接校准数据。估算方法:

1. **域缩放因子**:用两个已在三个域都校准过的DeepSeek模型,计算retail、telecom相对airline的
   token量比例(分别对prompt和completion token计算,flash与pro两个模型的比例取平均,作为"通用"
   域缩放因子,假设不同模型在不同域上的相对难度/对话长度变化趋势相近):
   - retail相对airline: prompt ×0.968, completion ×0.740
   - telecom相对airline: prompt ×1.505, completion ×0.625
   - (计算依据: flash的airline/retail/telecom平均prompt token分别为63973/71834/87232,
     平均completion token分别为3999/3362/2008;pro对应为48427/39340/79714和2915/1862/2179)
2. **Qwen模型的airline基线**:直接用任务2实测的单任务(prompt, completion)token数,再乘以上述
   域缩放因子得到retail/telecom估算值。
3. **qwen3-32b的特殊处理**:该模型任务2验证时那一次任务模型未能在合理步数内完成任务
   (reward=0,completion token仅209,明显是"提前放弃/被截断"的异常短值,不能代表该模型正常应答的
   token消耗)。故该模型的completion token改用同档位("非思考和思考模式"共用一档定价的qwen-open
   系列:27b/35b-a3b/122b-a10b/397b-a17b)四个模型的airline实测completion token均值
   (≈3322)替代,prompt token仍用实测值(31999,prompt受任务是否做完影响较小)。
4. **思考模型的completion token放大**:qwen3-235b-a22b-thinking-2507是"仅思考模式"模型,只有
   airline的1个任务样本,思维链长度对任务复杂度非常敏感,用同一因子外推到retail/telecom风险较高。
   按任务要求,对该模型在retail/telecom的completion token在域缩放之后再额外乘以**3倍**做保守粗估
   (未做外推的airline域不乘)。其余10个模型(含"非思考和思考模式"共享一档定价的qwen3.7-max/plus、
   qwen3.6-flash、qwen3.5系列、qwen3-32b,以及"仅非思考模式"的qwen3-235b-instruct)均不做此项放大,
   因为它们的输出价本身就是单一档(要么两种模式同价,要么模型本身不支持思考),任务2实测的completion
   token已经反映真实响应长度。
5. 汇率:1 USD = 6.8067 CNY(2026-07-01 CFETS中间价,见models.yaml)。

**局限性(如实说明)**:
- 仅DeepSeek两个模型的估算基于真实10任务均值,其余10个Qwen模型的retail/telecom估算基于单任务
  外推,存在较大不确定性(单任务样本方差未知,域缩放因子借用了DeepSeek系列的比例,不同模型家族/
  参数规模的实际域敏感度可能不同)。
- qwen3.6-flash/qwen3.5-35b-a3b/qwen3.7-plus为阶梯计费,若实际单任务prompt超过第一档上限
  (256K或128K token),会按更贵的第二档计费,本预估全部按第一档计算,存在轻微低估可能(τ2-bench
  单任务通常在数万到十几万token量级,大概率落在第一档,但telecom域的长对话有可能逼近该门槛)。
- 该预估**不包含** BFCL(`status: pending_integration`,尚未确定Qwen接入方式)和
  `thinking_budget_sweep`(思考预算扫描,见第4节单独估算)。

token,约为qwen3.7-max的1/80~1/100),总成本仅$9.64。

## 4. 补充:thinking_budget_sweep 粗估(不在主矩阵总成本内)

`analysis/models.yaml` 里的 `thinking_budget_sweep`(qwen3.6-flash,4个思考预算值:
0/1024/4096/16384,单域airline,50个任务,1个trial)是主矩阵之外的单独实验。粗估方法:沿用
qwen3.6-flash在airline域任务2实测的token均值(prompt=33450, completion=3417)作为4个budget档位
共同的近似基线(实际上不同思考预算会显著改变completion token量,尤其budget=0时应远小于其余三档,
本估算未按budget分别校准,只是给一个数量级参考):

- 单任务成本 ≈ (33450/1e6×1.2 + 3417/1e6×7.2) / 6.8067 ≈ $0.0095
- 4个budget档 × 50任务 × 1trial = 200次调用
- 粗估总成本 ≈ **$1.90 / ¥12.95**
