# 研究方案:预算受控下的开放权重LLM智能体评测

**工作标题(中)**:预算受控下的开放权重大模型智能体评测:成功率-预算曲线、价格反转与成本感知模型选择
**工作标题(英)**:Budget-Constrained Evaluation of Open-Weight LLM Agents: Success–Budget Curves, Price Reversal, and Cost-Aware Model Selection
**日期**:2026-07-04 | **状态**:方案定稿待确认

---

## 1. 研究问题(RQs)

- **RQ1**:在多步智能体任务上,给定每任务硬预算上限 B,各开放权重模型的成功率 S(B) 曲线形态如何?排行榜在不同预算档位下是否发生名次反转(rank crossover)?
- **RQ2**:单轮任务上已证实的"价格反转"现象(标价便宜的推理模型实际更贵,因思考token异质性)在多步智能体任务中是被放大还是被抹平?
- **RQ3**:显式告知智能体剩余预算(budget-aware)相比不告知(budget-unaware),能否改变 S(B) 曲线?
- **RQ4**:同一模型下扫描思考预算(利用Qwen3原生thinking budget控制),智能体任务成功率对思考预算的弹性如何?

## 2. 定位:与五个最近邻工作的差异(逐条可防守)

| 竞品 | 它做了什么 | 它没做什么(=我们的空间) |
|---|---|---|
| HAL(Princeton, ICLR 2026, arXiv 2510.11977) | 9闭源前沿模型×9基准,事后记录成本,$40K | 无预算约束协议、无S(B)曲线、**完全未覆盖DeepSeek/Qwen/GLM/Kimi等开放权重模型** |
| Cost-of-Pass(arXiv 2504.13359) | 经济学指标:每个正确解的期望成本 | 以单轮任务为主,无硬预算上限,无多步智能体 |
| Price Reversal(arXiv 2603.23971) | 单轮推理任务上发现21.8%配对出现价格反转 | **明确承认未做多步智能体任务**,并呼吁"把推理成本作为一级评测维度"——我们直接回应此呼吁 |
| BCAS(arXiv 2603.08877) | 预算受控的检索型QA智能体设计消融 | 仅限搜索问答一种支架,非跨模型系统评测 |
| Budget-Aware LLM Agents(2026) | 评测智能体预算意识+可训练成本控制 | 方法论文,非系统性多模型评测协议 |

**一句话定位**:HAL回答"不限成本谁最强",我们回答"**每任务预算¥X时该选谁**"——这是预算敏感部署者(也是开放权重模型的真实用户群)每天面对的问题,现有工作没有一个正面回答。

## 3. 核心方法设计

### 3.1 预算受控评测协议
- 成本记账:统一按 token 计价(输入/输出/思考分列)+ 按公开牌价折算货币成本;同时报告步数与墙钟时间。
- 硬预算上限:任务执行中累计成本达到 B 即截断,按当前状态判定成败。
- **关键方法学技巧(省钱10倍)**:对 budget-unaware 智能体,预算 B 下的轨迹 = 最大预算轨迹在 B 处的截断(因为智能体行为不依赖它不知道的预算)。故**每任务只需在最高预算下跑一次**,即可离线导出全部预算档位的 S(B) 曲线。budget-aware 条件(RQ3)需单独跑一遍。
- 每配置重复 k=3 次以估计方差(思考token可变性单查询可达9.7倍,方差报告是硬要求)。

### 3.2 指标定义(论文的公式贡献)
- **S(B)**:预算-成功率曲线(逐任务成本截断后聚合)。
- **AUBC**:预算-成功率曲线下面积(归一化),综合性价比单值指标。
- **B@τ**:达到成功率 τ 所需最小预算(可行性阈值指标)。
- **预算弹性 ε(B)**:dlogS/dlogB,刻画"再加一块钱换多少成功率"。
- **反转点集合**:任意两模型 S(B) 曲线的交点,输出"预算区间→最优模型"查找表(论文的实用卖点)。
- 统计:任务级配对bootstrap置信区间 + 不同预算档位下排名的Kendall τ稳定性。

### 3.3 基准选择(纯API可跑、无重型基础设施)
1. **τ-bench(airline + retail)**:客服工具调用智能体,HAL同款,多步、有确定性判分。
2. **BFCL v3 多轮**:函数调用,轻量,判分严格。
3. **ALFWorld**:文本具身环境,本地免费运行,长视野。
(备选扩展:SWE-bench Verified Mini,需本地Docker,作为审稿加分项视进度决定。)

### 3.4 模型矩阵(全部可经国内API获取)
- DeepSeek:V3.2(chat)、R1/reasoner 模式
- Qwen3 系列:8B / 32B / 235B,thinking on/off;**其中一个型号做思考预算扫描(RQ4)**
- GLM-4.x、Kimi K2.5、MiniMax-M2.5(经SiliconFlow等聚合API)
- 合计约 10-12 个模型配置 × 3 基准 × k=3

### 3.5 成本估算
最高预算单次跑法(3.1的截断技巧)下,估计总API开销 **¥800–2500(约$110–350)**,在用户预算内。先跑冒烟测试(每基准10任务×2模型)校准估算再全量。

## 4. 预期贡献(写进论文引言的三条)
1. 首个面向**开放权重/国产API模型**的多步智能体预算受控评测协议与公开结果(填补HAL明确留下的覆盖空白)。
2. 一组预算条件化指标(S(B)/AUBC/B@τ/弹性/反转点),把"给定预算选模型"变成可查表的工程决策。
